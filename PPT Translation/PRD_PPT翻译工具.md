**状态**：草稿

## 一、产品概述

### 背景与目标

跨语言场景下，PPT 的人工翻译效率低、格式还原成本高，现有通用翻译工具无法保留幻灯片的视觉布局，且术语精准度不足以应对专业场景。

本产品采用 **Web 应用 + 后端 API** 形态交付，核心目标：

1. 提供开箱即用的 Web 界面，任何用户无需安装即可完成 PPT 翻译；
2. 同时暴露 REST API，支持企业系统与开发者程序化集成；
3. 自动保留原始 PPT 布局与格式，支持场景预设保证专业术语准确性。

### 产品架构

```
用户（浏览器）
    │  拖拽上传 / 参数配置 / 进度查看 / 文件下载
    ▼
Web 前端（单页应用）
    │  HTTP / REST
    ▼
后端 API（FastAPI）
    ├── PPT 解析与格式还原（python-pptx）
    ├── 翻译引擎调用（DeepL / OpenAI / Google）
    └── 任务队列（异步翻译）
```

### 用户群体

| 类型 | 使用方式 | 代表场景 |
|------|---------|---------|
| 普通用户 | Web 界面 | 个人学习、工作汇报、出行演讲等日常场景 |
| 专业服务商 | Web 界面 / API | 翻译公司、会议服务商批量处理 |
| 企业研发团队 | REST API | 集成至内部文档管理、OA 或内容平台 |
| 个人开发者 | REST API | 嵌入个人工具或自动化脚本 |

## 二、核心功能需求

### F1 · Web 前端

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F1-1 | 提供拖拽上传区域，支持点击选择文件，接受 `.pptx` 格式 | P0 |
| F1-2 | 上传前展示文件名、页数、大小预览 | P0 |
| F1-3 | 提供语言方向选择（中→英 / 英→中）和翻译场景下拉选择 | P0 |
| F1-4 | 翻译过程中显示进度条与预计剩余时间，禁止重复提交 | P0 |
| F1-5 | 翻译完成后提供**一键下载**按钮，文件名格式：`原文件名_translated.pptx` | P0 |
| F1-6 | 错误状态友好提示（文件过大、格式不支持、翻译失败等） | P0 |
| F1-7 | 支持**历史记录**：展示本次会话内已翻译文件列表，可重新下载 | P1 |
| F1-8 | 支持**双语对照预览**：翻译完成后在页面内预览幻灯片缩略图 | P2 |

### F2 · 文件处理

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F2-1 | 支持上传 `.pptx` 格式文件，单文件最大 **50 MB**，最多 **500 张**幻灯片 | P0 |
| F2-2 | 翻译完成后返回同格式 `.pptx` 文件，Web 端直接下载，API 端支持回调 URL 推送 | P0 |
| F2-3 | 支持异步模式：提交后返回 `task_id`，前端轮询进度，API 端支持 Webhook | P0 |
| F2-4 | 支持批量翻译（Web 端多文件上传，API 端单次请求最多 10 个文件） | P1 |

### F3 · 翻译能力

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F3-1 | 支持 **中→英** 与 **英→中** 双向翻译 | P0 |
| F3-2 | 接入至少一种主流翻译引擎（DeepL / OpenAI / Google Translate），引擎可配置 | P0 |
| F3-3 | 支持**翻译场景（domain）** 参数，内置场景：通用、商务、医疗、法律、金融、学术、科技 | P0 |
| F3-4 | 支持**自定义术语表**（glossary），键值对形式，覆盖引擎默认翻译 | P1 |
| F3-5 | 支持**不翻译列表**，如品牌名、专有名词跳过翻译 | P1 |
| F3-6 | **翻译记忆**：相同文本片段复用已翻译结果，提升一致性与速度 | P2 |

### F4 · 格式保留

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F4-1 | 保留所有文本框的位置、尺寸、旋转角度 | P0 |
| F4-2 | 保留字体族、字号、加粗、斜体、颜色、下划线等字符格式 | P0 |
| F4-3 | 保留段落对齐方式、行间距、缩进 | P0 |
| F4-4 | 图片、形状、图表、背景、动画、备注页**原样保留，不做翻译** | P0 |
| F4-5 | 翻译后文本若超出原文本框范围，自动等比缩小字号以适配（可关闭） | P1 |
| F4-6 | 支持**双语对照模式**：在原文下方插入译文（颜色或字号有别） | P2 |

### F5 · API 接口规范

> Web 前端与外部调用方共用同一套后端 API。

#### 5.1 文件翻译（异步，推荐）

```
POST /v1/translate
Content-Type: multipart/form-data

参数：
  file         .pptx 文件（必填）
  source_lang  源语言，zh / en（必填）
  target_lang  目标语言，zh / en（必填）
  domain       翻译场景，默认 general（可选）
  engine       翻译引擎，默认 deepl（可选）
  glossary_id  术语表 ID（可选）
  font_shrink  字号缩小适配，true/false，默认 true（可选）
  webhook_url  翻译完成后回调地址（可选，API 调用方使用）

返回：
  { "task_id": "xxx", "status": "queued", "estimated_seconds": 30 }
```

#### 5.2 任务状态查询 / 结果下载

```
GET /v1/translate/{task_id}
返回：
  进行中：{ "task_id": "xxx", "status": "processing", "progress": 0.6 }
  完成：  { "task_id": "xxx", "status": "completed", "download_url": "...", "expires_at": "..." }
  失败：  { "task_id": "xxx", "status": "failed", "error": { "code": "...", "message": "..." } }

GET /v1/translate/{task_id}/download
返回：.pptx 文件流（直接下载）
```

#### 5.3 术语表管理

```
POST   /v1/glossaries          创建术语表
GET    /v1/glossaries/{id}     获取术语表
PUT    /v1/glossaries/{id}     更新术语表
DELETE /v1/glossaries/{id}     删除术语表
```

#### 5.4 翻译场景查询

```
GET /v1/domains    返回所有支持的翻译场景列表及说明
```

### F6 · 安全与鉴权

| 编号 | 需求 | 优先级 |
|------|------|--------|
| F6-1 | Web 用户：无需注册即可使用（匿名模式），通过 Session Cookie 标识会话 | P0 |
| F6-2 | API 调用方：通过 **API Key**（Bearer Token）鉴权 | P0 |
| F6-3 | 上传文件仅在服务端短暂存储，翻译完成后 **24 小时内自动删除** | P0 |
| F6-4 | 传输层强制 HTTPS | P0 |
| F6-5 | 匿名用户速率限制：每 IP 每日最多翻译 **5 个文件** | P1 |
| F6-6 | API Key 可配置速率限制（RPM / 每日文件数） | P1 |

## 三、非功能需求

| 类别 | 指标 |
|------|------|
| 性能 | 10 页以内 PPT ≤ 15s 完成翻译；超 10 页异步处理，前端实时显示进度 |
| 可用性 | 服务可用率 ≥ 99.5%（月度） |
| 格式还原率 | 翻译后布局与原文视觉一致率 ≥ 95%（内部测试集） |
| 多引擎容灾 | 主引擎不可用时自动 fallback 至备用引擎，用户无感知 |
| 兼容性 | Web 端支持 Chrome / Edge / Safari 最新两个主版本 |
| 错误响应 | 所有 API 错误返回标准 JSON：`{ "code": "...", "message": "..." }` |

## 四、翻译场景说明

| 场景值 | 说明 | 典型术语示例 |
|--------|------|------------|

## 五、优先级规划

### MVP（P0 全量）

- **Web 界面**：拖拽上传、语言/场景选择、进度显示、一键下载、错误提示
- 中英双向翻译 + 场景参数 + 至少一种翻译引擎接入
- 完整格式保留（文本框位置、字体、段落样式）
- 异步任务模式（前端轮询进度）
- 匿名访问（无需注册）+ HTTPS + 文件 24h 自动删除

### V1.1（P1）

- 批量翻译（Web 多文件 + API 批量接口）
- 自定义术语表 CRUD
- 不翻译列表
- 匿名用户速率限制 + API Key 管理
- 字号自动缩放适配
- 会话内历史记录

### V2.0（P2）

- 双语对照模式（输出 + 页面预览）
- 翻译记忆
- 多引擎 fallback
- 使用量统计 Dashboard（API 调用方）

## 六、关键约束与风险

| 风险 | 说明 | 应对 |
|------|------|------|
| 第三方 API 成本 | 按字符计费，大文件成本较高 | 翻译记忆复用 + 匿名用户限额 |
| 复杂格式还原 | 嵌套表格、艺术字、SmartArt 还原难度高 | MVP 阶段保留原始元素不翻译 |
| 中文字体缺失 | 服务端无中文字体导致乱码 | 内置常用中文字体或嵌入字体策略 |
| 引擎术语精度 | 通用引擎在专业领域准确率不稳定 | 术语表覆盖 + 场景 prompt 调优 |
| 大文件超时 | 页数多时翻译耗时长，同步接口易超时 | 强制异步模式 + 前端进度轮询 |

---

## 七、验收标准

1. **格式**：随机抽取 20 份测试 PPT，翻译后布局偏差 ≤ 5%；
2. **术语**：每个 domain 构造 50 条专业术语测试集，准确率 ≥ 90%；
3. **性能**：10 页 PPT 在 P95 下翻译完成 ≤ 15s；
4. **Web 体验**：上传到可下载全流程，用户操作步骤 ≤ 4 步；
5. **API**：所有接口按规范返回，错误码完备，通过 Postman 回归测试；
6. **安全**：匿名用户超出限额返回 429，文件 24h 后确认删除，API Key 缺失返回 401。
